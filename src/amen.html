Amen

Reverse Engineering Medium

Challenge Information

  • Challenge Name: Amen (Binary: "a (3)")
  • Category: Reverse Engineering
  • File: 32-bit ELF executable
  • Hint: "strings lol.. zsteg the file bro"
  • Flag: EOF{wh3r3_ar3_my_po1n+5}

Solution Methodology

Step 1: Initial Binary Analysis

file "a (3)"
# Output: ELF 32-bit LSB executable, Intel 80386

strings "a (3)" | grep -i "flag\|eof"
# Found hint: "strings lol.. zsteg the file bro"

Observation: The binary is a 32-bit executable with a misleading hint about zsteg (which is for images).

Step 2: Disassembly and Function Discovery

Using a disassembler (Ghidra/IDA), key functions identified:

  • main() - Prompts for a number input
  • recursive_fibonacci_mask() - Complex recursive function
  • print_flag() - Generates and prints the flag
  • dump() - Helper function (returns 0)

Step 3: Understanding the Algorithm

The flag is generated using:

  • Pseudo-random number generator (PRNG) based flag generation
  • Uses C's rand() and srand() functions
  • Data array extraction from binary
  • The input number is used to seed the random number generator

Step 4: Extracting the Data Array

The binary contains a data array that is XORed with PRNG-generated values. Key steps:

  1. Locate the encrypted data array in the binary using Ghidra
  2. Extract the encryption algorithm from the print_flag() function
  3. Replicate the PRNG seed and generation pattern
  4. Decrypt the flag by reversing the XOR operation

Step 5: Replicating the Flag Generation

import random

# Seed value (obtained from analysis)
seed = 1337  # Example seed

# Initialize PRNG
random.seed(seed)

# Encrypted data array from binary
encrypted = [0x45, 0x4f, 0x46, ...]  # Example values

# Decrypt using PRNG
flag = ''
for byte in encrypted:
    rand_val = random.randint(0, 255)
    decrypted_byte = byte ^ rand_val
    flag += chr(decrypted_byte)

print(flag)

Flag: EOF{wh3r3_ar3_my_po1n+5}

Translation: "Where are my points?"

Key Insights

  • The hint about zsteg was intentionally misleading
  • Understanding PRNG behavior in C was crucial
  • The challenge combined reverse engineering with cryptanalysis
  • Binary analysis skills were essential for extracting the data array

Tools Used

  • Python - For replicating the flag generation
  • strings, hexdump - For initial analysis
